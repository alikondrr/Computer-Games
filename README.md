# Computer-Games
## Описание проекта

Приложение WPF для поиска, фильтрации и управления каталогом компьютерных игр. Поддерживает работу с играми разных типов (одиночные, мультиплеерные, онлайн) на различных платформах (PC, консоль, мобильные устройства). Данные сохраняются в формате JSON или XML.

## Структура проекта

- **ComputerGames** — основной проект с пользовательским интерфейсом (WPF)
- **Model** — библиотека классов с бизнес-логикой
  - **Core/** — основные классы: Game, GameCatalog, интерфейсы платформ, статистика
  - **Data/** — сериализация данных (JSON, XML)

## Главное окно

| Элемент | Тип | Назначение | Когда активен |
|---------|-----|-----------|---------------|
| **Заголовок** | TextBlock | Отображает "Каталог компьютерных игр" | Всегда |
| **Открыть каталог** | Button | Открывает окно редактирования каталога с таблицей всех игр | Всегда |
| **Выберите игру** | ComboBox | Выпадающий список игр с учётом фильтров | Всегда |
| **Показать данные** | Button | Открывает окно с подробной информацией об игре | Только после выбора игры в списке |
| **Платформа** | ComboBox | Фильтр: Все / PC / console / mobile | Всегда |
| **Режим игры** | ComboBox | Фильтр: Все / single / multi / online | Всегда |
| **Формат** | ComboBox | Выбор формата сохранения: JSON / XML | Всегда |
| **Статус** | TextBlock | Показывает количество загруженных игр и статус операций | Всегда |

### Как работают фильтры

1. По умолчанию в обоих фильтрах выбрано **"Все"** — отображаются все игры
2. При выборе платформы и/или режима список игр обновляется автоматически
3. Фильтры работают по принципу **И** (если выбраны оба — показываются игры, удовлетворяющие обоим условиям)
4. Список всегда сортируется по названию в алфавитном порядке
5. При смене фильтров кнопка **"Показать данные"** становится неактивной до выбора игры

### Как работает смена формата

- По умолчанию данные загружаются из **JSON**
- При переключении формата в комбобоксе данные **автоматически сохраняются** в новый файл
- Файлы хранятся в папке `Документы/ComputerGamesData/`

## Окно информации об игре

**Как открыть:** выбрать игру в главном окне → нажать "Показать данные"

| Элемент | Тип | Назначение |
|---------|-----|-----------|
| **Название** | TextBlock | Полное название игры |
| **Жанр** | TextBlock | Жанр игры |
| **Возрастной рейтинг** | TextBlock | Возрастное ограничение |
| **Дата выхода** | TextBlock | Дата релиза |
| **Рейтинг качества** | TextBlock | Оценка качества (по 10-балльной шкале) |
| **Закрыть** | Button | Закрывает окно и возвращает в главное меню |

---

## Окно редактирования каталога

**Как открыть:** нажать "Открыть каталог" в главном окне

| Элемент | Тип | Назначение | Когда активен |
|---------|-----|-----------|---------------|
| **Таблица игр** | DataGrid | Список всех игр (название, жанр, возраст, рейтинг) | Всегда |
| **Поле ввода** | TextBox | Ввод данных новой игры | Всегда |
| **Добавить** | Button | Добавляет новую игру в каталог | Всегда |
| **Удалить** | Button | Удаляет выбранную игру | Только если игра выбрана в таблице |

### Как добавить игру

Введите данные в поле через запятую в формате:
тип, название, жанр, возраст, рейтинг

text

**Пример:**
single, Stardew Valley, Simulation, 6, 9.1

text

**Типы игр:**
- `single` — одиночная игра
- `multi` — мультиплеерная игра
- `online` — онлайн-игра

### Как удалить игру

1. Выберите игру в таблице (кликните по строке)
2. Нажмите **"Удалить"**
3. Подтвердите удаление в появившемся окне

После закрытия окна каталога список игр в главном окне обновляется автоматически.

---

## Форматы сохранения

- **JSON** — формат по умолчанию, файл `games.json`
- **XML** — альтернативный формат, файл `games.xml`

При переключении формата данные автоматически копируются в соответствующий файл.

---

## Первый запуск

При первом запуске приложения, если файл с данными не найден, автоматически создаётся тестовый каталог из 10 игр разных типов и жанров.

---

## Источники

Тестовые данные сгенерированы программно. Изображения для обложек игр не используются в данной версии приложения.
